Introduction

The address of the MPIS document is identified at the header of the first page, which indicates the discipline type "All disciplines" and the document type "Administration Instructions" the first revision 06 Feb 2002, revised at 01 Jan 2003, and 11 March 2003.

In order to manage the documents produced in MPIS it is necessary to give each one a unique identifying number. Following are the standard abbreviations, which have been adopted and will be used as a code numbers for MPIS documents.

Also the document number is created to facilitate various sorting and retrieval processes i.e. to help people how to store any MPIS document or retrieve it from its right location in the binder.

Proposed Codes for Modifications

Each document will be reviewed on a predetermined frequency. There may be revisions required as a result of these reviews. There are also occasions when the requirement for revisions are discovered when a document is applied in a new location. There must be a code to identify the number of revisions which have been made to a document, for example r0 = original approved document, r1 = first revision after approval etc.

Proposed Coding for MPIS Documents

As mentioned in the introduction the coding system applied to MPIS documents is intended to provide important information about the document to facilitate various sorting and retrieval processes.

The first 2 or 3 letters will indicate the equipment type, the second set of numbers will indicate the sequence number, the final 2 numbers indicate the number of times the document has been reviewed/revised, and we will add small letter ( a ) to indicate the language of the document is Arabic.
